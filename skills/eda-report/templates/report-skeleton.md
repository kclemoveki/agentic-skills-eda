# Report Skeleton — `/eda-report`

This is the literal markdown skeleton the skill writes to `reports/eda_report_<dataset_stem>.md`. The skill reads this template at Step 4 and fills the placeholders with content extracted from the notebook.

Use Spanish headings if the notebook narrative is in Spanish; English otherwise.

## Adaptation guidelines

- **No temporal columns?** In section 3, omit references to time period. In section 4, no temporal findings.
- **No statistical tests in the notebook?** Findings in section 4 should still cite numbers, but with descriptive language ("el 60% de los goles ocurren en la segunda mitad") rather than test results.
- **Modeling section absent in notebook?** Skip any modeling-derived finding in section 4.
- **Very small dataset (N < 100)?** Section 5 (Limitations) gets a bullet about sample size limiting inference. Section 4 findings frame as descriptive, not inferential.
- **Quality report shows 🔴?** Section 3 calls it out explicitly. Section 5 mentions data quality as a primary risk.
- **Notebook had injection-detection cells (e.g., `flag_injection`)?** Mention in section 5 as a quirk worth flagging to stakeholders ("se detectaron N filas con texto adversarial; fueron tratadas como ruido").
