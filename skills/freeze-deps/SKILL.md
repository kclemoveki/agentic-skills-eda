# Skill: Freeze Deps

Pin the dependencies of the notebook at `$ARGUMENTS` so the analysis can be reproduced. Adapts to the project's package manager: emits `requirements.txt` for pip projects and a `poetry add` command for Poetry projects.

## Step 1 — Detect the project's package manager

Look at the current working directory for these signals:

- `pyproject.toml` exists AND contains `[tool.poetry]` → **Poetry project**
- `pyproject.toml` exists with `[project]` table (PEP 621) but no `[tool.poetry]` → **PEP 621 project** (treat as pip-compatible — emit requirements.txt)
- No `pyproject.toml` → **pip project** → emit requirements.txt

This decision drives Step 4. Capture the choice for the report.

## Step 2 — Extract imports from the notebook

Run a Python one-shot via `Bash` that:

1. Loads the `.ipynb` as JSON.
2. Iterates over `cell_type == "code"` cells.
3. Joins the `source` lines per cell and parses each with `ast.parse` (catching `SyntaxError` and skipping the cell with a warning — IPython magics like `%load_ext` are not valid Python and will fail to parse).
4. Walks the AST collecting `ast.Import` and `ast.ImportFrom` nodes.
5. Extracts the **top-level package name** for each import:
   - `import sklearn.cluster` → `sklearn`
   - `from scipy.stats import ttest_ind` → `scipy`
   - `import pandas as pd` → `pandas`
6. Filters out modules from the standard library using `sys.stdlib_module_names` (Python 3.10+).
7. Maps import names to distribution names where they differ (use `importlib.metadata.packages_distributions()`):
   - `sklearn` → `scikit-learn`
   - `cv2` → `opencv-python`
   - `PIL` → `Pillow`

Returns a sorted list of unique distribution names.

## Step 3 — Resolve installed versions

For each distribution name, call `importlib.metadata.version(name)`. If a package is imported by the notebook but not installed in the current environment, include it in the output with a clear marker:

```
# pandas==2.1.0
# WARNING: <pkg> imported but not installed in current environment
```

Do not fail the whole skill on missing packages — emit the warning and continue.

## Step 4 — Emit the dependencies file

### If pip project

Write `requirements.txt` to the current working directory with one pin per line:

```
matplotlib==3.8.2
numpy==1.26.4
pandas==2.1.0
scipy==1.11.4
seaborn==0.13.0
scikit-learn==1.3.2
```

Sort alphabetically. Use exact pins (`==`), not ranges, for reproducibility.

### If Poetry project

Write `poetry_deps.txt` with two parts:

```
# Detected dependencies for <notebook>
# Install with:

poetry add 'matplotlib==3.8.2' 'numpy==1.26.4' 'pandas==2.1.0' 'scipy==1.11.4' 'seaborn==0.13.0' 'scikit-learn==1.3.2'
```

The user runs the command verbatim. We do **not** modify `pyproject.toml` directly — that risks corrupting existing TOML structure.

## Step 5 — Append a watermark cell to the notebook

Add a new code cell at the end of the notebook with:

```python
# Environment audit — generated by /freeze-deps
%load_ext watermark
%watermark -v -m -p <comma-separated package list>
```

Replace `<comma-separated package list>` with the actual list of detected packages. This cell is appended only if no cell with the comment `Environment audit — generated by /freeze-deps` already exists (idempotency check).

If the notebook already has the watermark cell, update its `%watermark -p` line with the current package list rather than appending a duplicate.

## Step 6 — Report to the user

Print exactly one line summarizing the result:

### Pip path
```
requirements.txt written (<N> packages) — watermark cell appended to <notebook>
```

### Poetry path
```
poetry_deps.txt written (<N> packages) — run the included `poetry add` command. Watermark cell appended to <notebook>
```

If any package was detected but not installed:
```
... — WARNING: <K> packages imported but not installed (see file)
```

## Constraints

- Python script must use type hints + NumPy-style docstrings.
- Never modify `pyproject.toml` or `poetry.lock` directly.
- Never modify the original cells of the notebook — only append (or update) the watermark cell.
- Be deterministic: same notebook + same environment → same output. Sorted lists, exact pins.
- Do not include the `watermark` package itself in the deps list unless it is actually imported elsewhere — it is a dev-time tool, not a runtime dep.
